# Share Module Design — KidWorksheets

> **Document Type:** Feature Architecture
> **Version:** 1.0
> **Last Updated:** 2026-07-23

---

## Overview

The Share Module allows both **Mentors** and **Children** to share worksheets in three distinct ways. The module is built using the **Strategy Design Pattern** — each share method is a separate, interchangeable strategy.

---

## Share Trigger Points

1. **Worksheet Detail Screen** — Share button in top-right menu or action bar
2. **Worksheet List** — Long-press on a card → Share option
3. **After Completion** — "Share this worksheet" on the results screen
4. **Child Dashboard** — Children can share worksheets they've accessed

## Strategy 1: Share to Google Drive

```
User taps "Google Drive"
    │
    ▼
Check Google Drive permission (OAuth scope: drive.file)
    │
    ├── Not authorized → Google OAuth popup (Drive scope)
    │
    ▼
Loading indicator ("Generating PDF...")
    │
    ▼
Mobile: Generate HTML from worksheet data
    │
    ▼
expo-print: HTML → PDF (local temp file)
    │
    ▼
POST /api/share/drive (multipart upload)
    │
    ▼
Backend:
  1. Receive PDF binary
  2. Upload to user's Google Drive via Drive API v3
     - Folder: "KidWorksheets" (created if not exists)
  3. Set sharing permissions (anyone with link can view)
  4. Return shareable Drive link
    │
    ▼
Mobile: Copy link to clipboard + show share sheet
    │
    ▼
User shares the Drive link (WhatsApp, SMS, Email, etc.)
    │
    ▼
Recipient opens link → Google Drive → Sees the PDF worksheet
```

## Strategy 2: Share as PDF File

```
User taps "PDF File"
    │
    ▼
Loading indicator ("Generating PDF...")
    │
    ▼
Mobile: Generate HTML from worksheet data
    │
    ▼
expo-print: HTML → PDF (local temp file at cache dir)
    │
    ▼
expo-sharing: Open native share sheet
    │
    ▼
Native OS Share Sheet appears with the PDF:
  - WhatsApp → Sends PDF as file
  - Gmail → PDF attached to email
  - Telegram → PDF file
  - Save to Files (iOS) / Downloads (Android)
  - Print (AirPrint / Cloud Print)
```

---

## Strategy 3: Share App Deep Link

```
User taps "Open in App"
    │
    ▼
POST /api/share/link (with worksheetId)
    │
    ▼
Backend:
  1. Check worksheet.isPublic === true (only public worksheets can be shared this way)
     OR generate time-limited access token for private worksheets
  2. Generate Firebase Dynamic Link:
     - Deep link: kidworksheets://worksheet/{worksheetId}
     - Fallback: App Store / Play Store URL
  3. Return short URL
    │
    ▼
Mobile: Show share sheet with the URL + copy button
    │
    ▼
Recipient taps link:
  - App installed → Opens app → Navigates to Worksheet screen
  - App not installed → Goes to App Store / Play Store → Installs → Opens worksheet
```

## PDF Generation Template

The PDF is generated from HTML using `expo-print`. The template includes:
- App logo + "KidWorksheets" branding
- Class name, Subject name
- Worksheet title
- Questions with options (formatted per question type)
- Footer with "Generated by KidWorksheets"
- **Blank answer lines** for IMAGE_BASED worksheets
- For INTERACTIVE worksheets: printable version with blank answer spaces

```typescript
// Pseudocode — PDF template function
function generateWorksheetHTML(worksheet: Worksheet, questions: Question[]): string {
  return `
    <html>
      <head>
        <style>
          /* Print-optimized CSS */
          body { font-family: 'Nunito', sans-serif; padding: 24px; }
          .header { display: flex; justify-content: space-between; }
          .question { margin-bottom: 20px; page-break-inside: avoid; }
          .mcq-option { margin-left: 20px; margin-top: 6px; }
          .answer-line { border-bottom: 1px solid #ccc; height: 30px; }
        </style>
      </head>
      <body>
        <div class="header">
          <h1>${worksheet.title}</h1>
          <p>Class: ${worksheet.class.name} | Subject: ${worksheet.subject.name}</p>
        </div>
        <hr/>
        ${questions.map(renderQuestion).join('')}
        <footer>Generated by KidWorksheets App</footer>
      </body>
    </html>
  `;
}
```

## Access Control for Sharing

| Worksheet Type | Strategy 1 (Drive) | Strategy 2 (PDF) | Strategy 3 (Deep Link) |
|---|---|---|---|
| Public (admin-approved) | ✅ Any user | ✅ Any user | ✅ Open to all |
| Private (mentor-created) | ✅ Mentor only | ✅ Mentor only | ⚠️ Time-limited token |
| Image-based (homework) | ✅ Mentor only | ✅ Mentor only | ❌ Not shareable |
| Child sharing | ✅ If allowed by mentor | ✅ If allowed by mentor | ✅ Public worksheets |
