# Phase 4: Sharing Module

> **Sprint Duration:** Week 10
> **Status:** 🔲 Not Started
> **Prerequisite:** Phase 2 complete ✅
> **Goal:** Full sharing capability — Google Drive, native PDF share, and in-app deep link.

---

## Objectives

1. PDF generation from worksheet data (Interactive + Image-based)
2. Google Drive API upload + shareable link
3. Native OS share sheet with PDF file
4. Firebase Dynamic Links for in-app deep linking
5. Share bottom sheet UI (used from any worksheet screen)

---

## Backend Checklist

### Share APIs
- [ ] `POST /api/share/drive` — receive PDF binary → upload to user's Google Drive → return Drive link
- [ ] `POST /api/share/link` — generate Firebase Dynamic Link for a worksheetId → return short URL
- [ ] `GET /api/worksheets/{id}/share-info` — returns: title, class, subject, thumbnailUrl (for share preview)

### Google Drive Integration (Backend)
- [ ] Google Drive API v3 client initialized (`googleapis` npm package)
- [ ] OAuth2 client using user's Google access token (passed from mobile)
- [ ] Create folder "KidWorksheets" in user's Drive (if not exists)
- [ ] Upload PDF to that folder
- [ ] Set permission: anyone with link → reader
- [ ] Return: `{ driveFileId, driveUrl, webViewLink }`

### Firebase Dynamic Links
- [ ] Firebase Dynamic Links configured in Firebase Console
- [ ] Domain: `kidworksheets.page.link` (or custom domain)
- [ ] Deep link format: `https://kidworksheets.app/worksheet/{id}`
- [ ] Fallback iOS: App Store link
- [ ] Fallback Android: Play Store link
- [ ] Handle incoming dynamic link on app open (Expo Router)

---

## Mobile Checklist

### PDF Generation
- [ ] `WorksheetPDFGenerator` utility using `expo-print`
- [ ] HTML template:
  - [ ] Header: app logo + "KidWorksheets" branding
  - [ ] Meta: Class, Subject, Title, Date
  - [ ] Questions rendered per type (MCQ options listed, blanks shown as underlines)
  - [ ] Footer: "Generated by KidWorksheets App"
- [ ] Save PDF to `expo-file-system` cache directory
- [ ] Cleanup after share is complete

### Share Bottom Sheet
- [ ] Custom bottom sheet component (react-native-bottom-sheet)
- [ ] 3 strategy cards with icon + title + description:
  - [ ] 📁 **Google Drive** — "Save to Drive & get a link"
  - [ ] 📄 **PDF File** — "Share as PDF via WhatsApp, Email, etc."
  - [ ] 🔗 **In-App Link** — "Share link to open in KidWorksheets app"
- [ ] Loading spinner inside each card while action is in progress
- [ ] Success state: "Copied to clipboard ✓" / "Shared!" toast
- [ ] Error state: retry button

### Strategy 1 — Google Drive
- [ ] Check: does mentor have Drive scope authorized?
  - [ ] No → trigger Google Sign-In with Drive scope addition
- [ ] Show loading: "Generating PDF..." → "Uploading to Drive..."
- [ ] `POST /api/share/drive` with PDF binary
- [ ] On success: show Drive link + "Copy Link" button + "Open in Drive" button

### Strategy 2 — PDF File
- [ ] Generate PDF locally
- [ ] `expo-sharing.shareAsync(pdfUri, { mimeType: 'application/pdf' })`
- [ ] Native share sheet opens automatically (WhatsApp, Gmail, etc.)
- [ ] No backend call needed

### Strategy 3 — In-App Deep Link
- [ ] `POST /api/share/link` with worksheetId
- [ ] On success: "Link copied! 📋" toast + native share sheet with URL text
- [ ] On recipient side: `expo-linking` handles the URL, navigates to worksheet

### Share Access Points
- [ ] Worksheet detail screen → share button (top-right)
- [ ] Worksheet list → long-press card → share
- [ ] Results screen → "Share this worksheet" button
- [ ] Child mode → share button (same bottom sheet, child can share too)
